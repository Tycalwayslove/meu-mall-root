# 飞书 Base 模板：H5 页面需求排期

## 用途

本文定义 MeuMall H5 需求排期多维表格的推荐表结构。创建或维护飞书 Base 时，以本文为字段事实源。

## Base 名称

建议名称：

```text
MeuMall H5 需求排期
```

## 表 1：H5 页面需求

### 用途

以单个 H5 页面作为主需求主体，承载页面范围、路由、状态、排期、负责人和子需求关联。

### 字段

| 字段名 | 字段类型建议 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 页面显示名称 | 文本 | 是 | 主显示字段，用于关联展示，例如 `推广首页｜/promotion`。 |
| 页面需求 ID | 文本 | 是 | 稳定编号，例如 `PAGE-HOME-001`。 |
| 页面名称 | 文本 | 是 | 页面中文名称。 |
| 路由 | 文本 | 是 | H5 路由，例如 `/promotion/benefits`。 |
| 页面层级 | 单选 | 是 | `一级`、`二级`、`三级`、`原生`、`待定`。 |
| 所属模块 | 单选 | 是 | `首页`、`推广`、`商品`、`我的`、`订单`、`通用`、`智能体`、`待定`。 |
| 端归属 | 单选 | 是 | `H5`、`App`、`Hybrid`、`待定`。 |
| 容器策略 | 单选 | 是 | `tab-root-webview`、`new-h5-webview`、`current-webview-push`、`native-page`、`native-modal`、`待定`。 |
| 开发状态 | 单选 | 是 | `未开始`、`需求澄清`、`设计确认`、`静态页面`、`开发中`、`待联调`、`联调中`、`可测试`、`可验收`、`已完成`、`阻塞`、`暂缓`。 |
| 优先级 | 单选 | 是 | `P0`、`P1`、`P2`、`P3`。 |
| H5 负责人 | 人员 | 否 | H5 负责人。 |
| 责任组 | 单选 | 是 | `前端`、`后端`、`原生`、`测试`、`iOS`、`Android`、`产品`、`设计`、`项目管理`、`待定`。 |
| 协作成员 | 关联 | 否 | 关联 `团队人员与角色`，真实人员未定时使用待分配占位。 |
| 后端负责人 | 人员 | 否 | 可由关联接口汇总，也可手填主负责人。 |
| 原生负责人 | 人员 | 否 | 可由关联原生需求汇总，也可手填主负责人。 |
| QA 负责人 | 人员 | 否 | 页面测试负责人。 |
| 计划开始 | 日期 | 否 | 页面计划开始日期。 |
| 计划完成 | 日期 | 否 | 页面计划完成日期。 |
| H5 工时 | 数字 | 否 | 单位：人日。 |
| 关联版本 | 关联 | 否 | 关联 `版本迭代排期`。 |
| 关联后端需求 | 关联 | 否 | 关联 `后端接口需求`。 |
| 关联原生需求 | 关联 | 否 | 关联 `原生 App 对接需求`。 |
| 关联测试需求 | 关联 | 否 | 关联 `测试验收需求`。 |
| 阻塞项 | 多行文本 | 否 | 当前阻塞原因。 |
| Figma 链接 | URL | 否 | 设计稿链接。 |
| 对接文档 | URL | 否 | 飞书知识库或仓库文档链接。 |
| 备注 | 多行文本 | 否 | 其它说明。 |

| 备注 | 多行文本 | 否 | 其它说明。 |

## 表 3：原生 App 对接需求

### 用途

记录 H5 与 iOS、Android 的 WebView 容器、Bridge、登录态、返回、切 Tab、支付、分享等对接事项。

### 字段

| 字段名 | 字段类型建议 | 是否必填 | 说明 |
| --- | --- | --- | --- |
| 原生显示名称 | 文本 | 是 | 主显示字段，用于关联展示，例如 `打开搜索 WebView｜L-006`。 |
| 原生需求 ID | 文本 | 是 | 稳定编号，例如 `APP-ROUTE-001`。 |
| 关联页面 | 关联 | 是 | 关联 `H5 页面需求`。 |
| 关联链路 | 文本 | 否 | 路由矩阵编号，例如 `L-018`。 |
| 平台 | 单选 | 是 | `双端`、`iOS`、`Android`、`待定`。 |
| 对接类型 | 单选 | 是 | `打开 H5 WebView`、`关闭 WebView`、`切 Tab`、`原生页面`、`支付`、`分享`、`保存图片`、`登录认证`、`Cookie 注入`、`状态栏`、`待定`。 |
| Bridge 能力 | 文本 | 否 | 如 `router.navigate`、`token_expired`。 |
| 容器策略 | 单选 | 是 | `tab-root-webview`、`new-h5-webview`、`current-webview-push`、`native-page`、`native-modal`、`待定`。 |
| H5 发起动作 | 多行文本 | 否 | H5 侧何时发起、传什么。 |
| App 接收动作 | 多行文本 | 否 | App 侧收到后做什么。 |
| iOS 状态 | 单选 | 是 | `待补充`、`待确认`、`已确认`、`开发中`、`自测完成`、`联调中`、`可测试`、`通过`、`阻塞`、`暂不需要`。 |
| Android 状态 | 单选 | 是 | 同上。 |
| 优先级 | 单选 | 是 | `P0`、`P1`、`P2`、`P3`。 |
| 责任组 | 单选 | 是 | 原生对接需求默认 `原生`，双端细分看 iOS/Android 状态。 |
| 协作成员 | 关联 | 否 | 关联 `团队人员与角色`，双端事项同时关联 iOS 和 Android 占位成员。 |
| 原生负责人 | 人员 | 否 | App 负责人。 |
| App 测试包版本 | 文本 | 否 | iOS/Android 测试包版本。 |
| 计划完成 | 日期 | 否 | 原生计划交付日期。 |
| 原生工时 | 数字 | 否 | 单位：人日。 |
| 对接文档 | URL | 否 | Bridge 或路由对接文档。 |
| 阻塞项 | 多行文本 | 否 | 当前阻塞原因。 |
| 备注 | 多行文本 | 否 | 其它说明。 |

## 表 6：团队人员与角色

### 用途

承载部门架构、人员占位和后续真实成员映射。真实人员未录入时，先用待分配占位保证所有任务都有可筛选的协作对象。

| 字段名 | 字段类型建议 | 是否必填 | 说明 |

| 人员/角色名称 | 文本 | 是 | 主显示字段，例如 `前端待分配`、`张三`。 |
| 部门/角色 | 单选 | 是 | `前端`、`后端`、`测试`、`iOS`、`Android`、`产品`、`设计`、`项目管理`、`待定`。 |
| 真实姓名 | 文本 | 否 | 真实人员姓名。 |
| 飞书 OpenID | 文本 | 否 | 后续如需要写入飞书人员字段，可用于映射。 |
| 负责模块 | 多选 | 否 | `首页`、`推广`、`商品`、`我的`、`订单`、`通用`、`智能体`、`待定`。 |
| 状态 | 单选 | 是 | `待补充`、`已确认`、`暂不参与`。 |
| 备注 | 多行文本 | 否 | 协作说明。 |

| 字段名 | 字段类型建议 | 是否必填 | 说明 |
| --- | --- | --- | --- |

| 任务类型 | 单选 | 是 | `H5 页面`、`后端接口`、`原生对接`、`测试验收`、`版本`。 |
| 来源需求 ID | 文本 | 是 | 来源表稳定编号，例如 `PAGE-PROMOTION-001`。 |
| 来源表 | 文本 | 是 | 任务来自哪张表。 |
| 关联页面 | 关联 | 否 | 关联 `H5 页面需求`。 |
| 责任组 | 单选 | 是 | `前端`、`后端`、`原生`、`测试`、`iOS`、`Android`、`产品`、`设计`、`项目管理`、`待定`。 |
| 协作成员 | 关联 | 否 | 关联 `团队人员与角色`。 |
| 当前状态 | 单选 | 是 | 统一状态口径，用于领导和项目负责人快速看进度。 |
| 优先级 | 单选 | 否 | `P0`、`P1`、`P2`、`P3`。 |
| 计划开始 | 日期 | 否 | 任务计划开始日期。 |

| 下一步动作 | 多行文本 | 否 | 下一步要做的具体动作。 |
| 备注 | 多行文本 | 否 | 其它说明。 |
