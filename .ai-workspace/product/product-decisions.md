# 喵呜产品决策记录

## 状态

active

## 说明

本文记录已经确认的产品事实。后续 AI 会话必须优先读取本文，不能再按普通电商默认假设推进。

## D001 主 Tab 结构

状态：已确认

主 Tab 为：

- 首页
- 智能体
- 推广
- 我的

影响：

- 原生 App 负责底部 Tab 容器。
- H5 负责首页、推广、我的中的大部分内容。
- 智能体 Tab 由 App 原生负责。

## D002 无购物车

状态：已确认

喵呜 APP 没有购物车概念。商品购买路径是：

```text
商品详情 -> 立即购买 -> 订单确认
```

影响：

- 不建立购物车 Tab。
- 不按传统购物车结算建模。
- 商品详情和订单确认是购买链路核心。
- 缓存规则中“购物车”只作为通用电商参考，不作为喵呜业务事实。

## D003 H5 页面都需要登录态

状态：已确认

H5 的所有页面都通过登录后携带 token 展示。每个接口都需要在 header 中传 token。

影响：

- H5 API client 必须支持统一 token 注入。
- H5 页面默认按登录态页面处理。
- token 体系后续单独建设。
- token 不得存入普通 H5 缓存。

## D004 私域平台只有登录，没有注册

状态：已确认

喵呜是私域平台。使用 App 的用户都是已经注册的用户，App 内只有登录，没有注册。

例外：运营投放链路可以使用外部 H5 注册页。该入口不是 App 内登录页的注册链路，不改变 App 内“只有登录”的产品事实。

影响：

- 登录页不设计注册链路。
- 未登录时引导登录，不引导注册。
- 用户身份由既有体系提供。
- 运营二维码注册入口固定为 `https://hybird.aigcpop.com/register`，由独立 Node resolver 解析 Java active manifest 后跳转到当前 active H5 版本注册页。

## D005 登录后即达人

状态：已确认

用户登录后即视为达人，不需要在 App 内申请成为达人。

影响：

- 推广和达人能力不需要“申请达人”流程。
- 达人等级、佣金、推广工具按登录用户展示。
- 权限差异主要来自达人等级或后台配置。

## D006 达人与会员是一套体系

状态：已确认

达人和会员在 UI 命名上可能不一致，但功能上属于同一套体系。

影响：

- 不拆两套独立等级模型。
- 达人等级、会员权益、佣金能力需要统一建模。
- 文案和 UI 可以有不同叫法，但数据和规则应保持一致。

## D007 佣金 N+1 计算并支持提现

状态：已确认

佣金按 N+1 日期计算前一天佣金。平台存在提现能力。

影响：

- 佣金不是完全实时最终值。
- 需要区分预计佣金、待结算佣金、已结算佣金和可提现余额。
- 提现、结算和扣回规则后续单独定义。

## D008 智能体不进入当前 H5 业务开发范围

状态：已确认

智能体功能已有老项目页面和代码。本项目当前只需要在 App 中展示 demo 壳子，不需要在 H5 中具体开发智能体功能。

影响：

- 智能体 Tab 归属 App。
- H5 不实现智能体首页、资料、创建、编辑等具体业务。
- 页面盘点中智能体相关内容只保留为端归属参考。

## D009 首页和消息入口属于 H5

状态：已确认

首页内容全部由 H5 实现。消息入口也属于 H5。

影响：

- 首页 banner、分类、活动、推荐商品由 H5 页面承载。
- 消息入口先按 H5 跳转处理。
- 后续如接入原生 IM，需要新增决策。

## D010 我的页面二级页面归属

状态：已确认

我的页面中，除设置外，其它二级页面暂定都是 H5。

影响：

- 收藏、会员/达人、收益、订单或记录类页面优先按 H5 建模。
- 设置页保留 App 原生或待定。

## D011 咨询入口暂不实现具体内容

状态：已确认

商品详情只提供咨询入口，暂不实现具体咨询或 IM 内容。

影响：

- 商品详情可以保留咨询按钮或入口。
- 暂不建设 IM 页面、消息协议和咨询数据模型。
- 后续接入咨询能力时再确认 App/H5 归属。

## D012 后台管理端需要预留业务配置扩展

状态：已确认

后台配置能力后续需要支持首页 banner、分类、活动信息等配置管理。该能力已迁移到外部 Java 体系，不再由本仓库的 `admin-meumall` / `server-meumall` 实现。

影响：

- H5 只消费 Java/API 或配置下发结果。
- 不再为 H5 需求修改 `admin-meumall` 或 `server-meumall`。
- 当前阶段先预留 H5 侧展示、fallback 和契约字段，不在本仓库实现管理后台。

## D013 当前 Figma 先按实验稿推进

状态：已确认

当前 Figma 画板暂时都按实验稿处理。先基于现有信息推进，后续逐页修改和确认正式页面。

影响：

- 页面盘点保持 draft。
- 不把任一具体画板视为最终视觉验收标准。
- 后续具体页面开发前仍需创建正式工作项。

## D014 运营外部注册 H5 使用固定公开入口

状态：已确认

运营二维码、外部邀请和私域拉新使用固定注册入口：

```text
https://hybird.aigcpop.com/register
```

该入口由独立 Node resolver 查询 Java active manifest 后 302 到当前 H5 版本下的 `/register` 页面，例如 `/h5-v/v1.0.28/register`。Java active manifest 使用 `GET /platform/h5Release/active`，不拼接 `environment` query。

影响：

- 运营不得直接投放 `/h5-v/<version>/register`。
- H5 发版 manifest 必须声明 `/register` route。
- 回滚或切换 active manifest 后，固定入口自动指向新的 active 版本。
- 该入口是外部注册 H5 例外，不代表 App 内登录页新增注册能力。

## D015 当前仓库只维护 H5 C 端

状态：已确认

当前仓库后续需求和修改只考虑 `hybird-meumall` H5 C 端。旧 `server-meumall`、`admin-meumall`、`app-meumall` 和 `meumall-ci` 已从当前工作区移除：H5 版本管理、active manifest、业务 API 和后台配置由外部 Java 体系承接；App/WebView 容器和 iOS 能力属于外部运行环境。

影响：

- AI 工作流不得再为普通 H5 需求恢复 `server-meumall`、`admin-meumall`、`app-meumall` 或 `meumall-ci`。
- 涉及接口时只更新 H5 侧调用、契约和对接说明，Java 实现作为外部依赖记录。
- 涉及 Bridge/WebView 时只实现 H5 侧调用、能力检测和 fallback，iOS/App 实现作为外部依赖记录。
- 根目录保留 H5 发布脚本、Nginx 模板和 `/register` resolver；旧 server/admin/app 项目目录、旧本地 Jenkins 工作区、旧 server/admin Docker 和测试服脚本不再维护。
