# 电商数据一致性规则

## 目的

本文件定义 MeuMall 电商业务数据的缓存和一致性等级。后续商品、购物车、订单、支付、优惠券、地址等业务开发必须先判断数据等级，再选择缓存、展示和交易策略。

## 数据等级

### 可缓存数据

这类数据可以缓存，但必须有 TTL、版本或失效策略。

示例：

- 首页模块。
- Banner。
- 分类树。
- 活动基础内容。
- 商品基础信息。
- 帮助文档。
- 最近浏览。
- 搜索历史。
- 表单草稿。

允许位置：

- CDN。
- Next.js 服务端缓存。
- Service Worker。
- IndexedDB。
- Native cache。

### 可快照展示但不可最终信任的数据

这类数据可以在弱网或无网时展示，但 UI 必须提示“缓存数据”或“上次更新于”。用户不能基于快照完成交易。

示例：

- 购物车摘要。
- 我的页面用户摘要。
- 最近订单摘要。
- 优惠券数量摘要。
- 商品上次展示价格。

允许位置：

- IndexedDB。
- Native cache。
- 内存缓存。

要求：

- 退出登录必须清理。
- 切换账号必须清理。
- 网络恢复后必须刷新。
- 交易操作必须重新请求真实接口。

### 不可缓存或不可离线承诺的数据

这类数据必须实时确认，不能作为离线可信数据。

示例：

- 支付状态。
- 下单结果。
- 库存最终状态。
- 价格最终状态。
- 余额。
- 订单最新状态。
- 优惠券最终可用性。
- 地址修改结果。
- 权限判定结果。

要求：

- API 必须 no-store 或等价策略。
- 不得被 CDN 缓存。
- 不得被 Service Worker Cache 缓存。
- 不得用缓存结果继续交易流程。
- 网络失败时必须阻止继续交易，并给出明确提示。

## 页面数据规则

### 首页和分类

可以展示公共缓存内容。弱网时可展示上次成功快照。网络恢复后刷新。

### 商品详情

商品标题、图片、描述、参数可以缓存。价格、库存、优惠、配送承诺和可购买状态必须实时确认。

### 购物车

可以展示购物车摘要快照，但必须标记为缓存数据。结算前必须重新拉取真实购物车、价格、库存和优惠。

### 我的

可以展示用户摘要快照，但必须标记为缓存数据。余额、优惠券可用性、订单最新状态必须实时确认。

### 订单

订单列表和订单详情不得共享缓存。弱网下可以展示最近订单摘要，但不能承诺最新状态。

### 支付和下单

必须实时请求。无网或接口失败时不得继续流程。

## 存储安全

禁止存入 H5 普通缓存：

- access token。
- refresh token。
- 支付凭证。
- 身份证、完整地址等敏感明文。
- 服务端密钥。

如需长期保存敏感信息，必须使用原生安全存储或项目确认的安全方案。

## 验收要求

涉及电商业务数据的工作项必须声明：

- 数据属于哪个一致性等级。
- 可以缓存在哪里。
- TTL 或失效规则是什么。
- 退出登录是否需要清理。
- 网络失败时用户能做什么。
- 交易前是否重新确认。
